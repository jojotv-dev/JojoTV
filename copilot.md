\# Règles pour GitHub Copilot



\## Langue

\- Toujours répondre en français dans toutes les suggestions, commentaires, explications, descriptions de commits et messages générés.



\## Règle fondamentale du projet

Toute modification, fonctionnalité, correction ou proposition doit être systématiquement appliquée aux trois layouts suivants :

\- modern

\- grid

\- classic



\### Exigences strictes

\- Ne jamais proposer une modification pour un seul layout.

\- Toujours produire les trois versions : modern + grid + classic.

\- Si un layout ne peut pas recevoir la modification, expliquer pourquoi et proposer une alternative.



\## Style de travail attendu

\- Harmoniser les trois layouts autant que possible.

\- Expliquer clairement les différences entre les versions modern, grid et classic.

\- Préserver la cohérence globale du projet.



\## Documentation

\- Documenter en français chaque version du layout.

\- Ajouter des commentaires clairs pour distinguer les trois implémentations.



\## Tests

\- Toute proposition de test doit couvrir les trois layouts.

\- Ajouter des tests spécifiques si un layout nécessite un comportement particulier.



\## Résumé pour Copilot

\- Toujours parler en français.

\- Toujours générer les modifications pour modern, grid et classic.

\- Ne jamais proposer une modification isolée.

\- Expliquer les différences entre les trois versions.
